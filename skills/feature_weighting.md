# Навык: Взвешивание характеристик

## Основной принцип
При выборе победителя акцент делается на характеристики, указанные в feature_priority пользователя.

## Алгоритм выбора
1. Выделить характеристики из feature_priority, которые есть в обоих товарах
2. Сравнить значения по каждой приоритетной характеристике:
   - Числовые: больше = лучше (кроме price, weight: меньше = лучше)
   - Категориальные: использовать онтологию из product_ontology.md
3. Подсчитать "победы" по приоритетным характеристикам
4. Если ничья по приоритетным — использовать второстепенные характеристики
5. Если всё равно ничья — объявить "Ничья" с рекомендацией по сценарию использования

## Обработка числовых характеристик
| Характеристика | Лучшее значение |
|---------------|----------------|
| battery (мАч, Вт·ч) | Больше |
| memory, storage, ram | Больше |
| camera (Мп) | Больше |
| price | Меньше |
| weight | Меньше |
| rating | Больше |

## Формулировка winner_reason
- Обязательно упомянуть топ-1 приоритетную характеристику, которая определила выбор
- Пример: "Победитель выбран по приоритету 'батарея': 5000 мАч > 3349 мАч"

## Особые случаи
- Если характеристика отсутствует у одного товара: считать её проигранной
- Если feature_priority пуст или не указан: использовать стандартное взвешивание (цена + рейтинг)
